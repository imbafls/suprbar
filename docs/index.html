What you get

Spend at a glance

Equivalent API cost from local JSONL, plus optional Anthropic Admin API for org-wide actuals.

Budgets & alerts

Daily, weekly, and monthly limits with tray color warnings when you approach the cap.

Local only

Reads ~/.claude/projects/**/*.jsonl directly. Nothing leaves your machine unless you enable the Admin API.

Multi-source (roadmap)

Provider plugin shape is in place — Cursor, Codex CLI, and more sources coming next.

Range filters

Today, 24h, 7d, week, month, 30d, 90d — server-side cache for snappy tab switches.

Install

Two paths — pick one.

# 1. Pre-built installer
Download suprbar-setup.exe from the GitHub Releases page, run it.

# 2. From source (Python 3.11+)
git clone https://github.com/imbafls/suprbar
cd suprbar
pip install -r requirements.txt
python -m suprbar

Add a data source

# suprbar/providers/cursor.py
def today_summary():
    return {
        "id": "cursor",
        "label": "Cursor · local",
        "ok": True,
        "cost_today": 0.0,
        "tokens_today": {"input": 0, "output": 0, ...},
        ...
    }

Register in aggregator.py and add a config toggle. Full extension guide.

Philosophy

Numbers you can act on. Spend, limits, burn rate — not theatre metrics.

Your data stays local. Zero phone-home. Admin API is opt-in only.

Multi-tool by design. One flyout, many sources — Claude Code today, more AIs tomorrow.

Extension over bloat. New tools = new provider module, not a settings explosion.
