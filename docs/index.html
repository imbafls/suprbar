A coach in your tray, not a counter.

supr.bar watches your local Claude Code sessions and surfaces specific observations — when you're iterating in circles, when you're winning, when to stop. No login. No telemetry. Just your data, observed.

What you get

Live observations

Six built-in rules watch your active session and surface notes when something changes — signal, not noise.

Session retros

When a session goes idle, supr.bar drops a 3-bullet markdown summary into ~/.suprbar/sessions/.

Local only

Reads ~/.claude/projects/**/*.jsonl directly. Nothing leaves your machine.

Write your own rule

Drop a 30-line Python file into ~/.suprbar/rules/. The coach picks it up at next launch.

Tiny config

Roughly a dozen settings. Themes, density, hotkeys, per-rule mute. That's it.

Install

Two paths — pick one.

# 1. Pre-built installer
Download suprbar-setup.exe from the GitHub Releases page, run it.

# 2. From source (Python 3.11+)
git clone https://github.com/omertaji/suprbar
cd suprbar
pip install -r requirements.txt
python -m suprbar

Write a rule in 30 lines

from suprbar.coach.rule import Observation, Rule


class LongMessage(Rule):
    id          = "long-message"
    name        = "Long prompt"
    description = "Nudges when the last message is over 4k chars."

    def evaluate(self, ctx):
        last = ctx.recent_user_messages(limit=1)
        if not last or len(last[0].text) < 4_000:
            return None
        return Observation(
            id=self.id, severity="info", confidence=0.7,
            title="Long prompt",
            body=f"Your last message was {len(last[0].text):,} chars. "
                 "Long prompts often hide multiple asks.",
            tip="Split into two turns: state the constraint, then the change.",
        )

Drop it in suprbar/coach/rules/ or ~/.suprbar/rules/. Restart. Full extension guide.

Philosophy

Observe, don't nag. Severity tops out at warn and never blocks.

Your data stays local. Zero phone-home. Anthropic Admin API integration is opt-in only.

Plain text wins. Session retros are markdown files in a folder.

Extension over configuration. The settings panel is intentionally small. If you need more, write a rule.
